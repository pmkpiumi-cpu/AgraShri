## 2. Security Assessment
Security has been integrated at the architectural level to protect student data and prevent system abuse.

*   **Server-Side Rate Limiting:** Implemented a custom rate-limiting mechanism in `src/app/actions/enroll.ts` that tracks IP addresses to prevent spam submissions (Max 3 requests per hour).
*   **Input Validation:** Every form field is strictly validated using **Zod** schemas before processing, preventing Cross-Site Scripting (XSS) and injection attacks.
*   **Environment Variable Protection:** Sensitive credentials (like EmailJS API keys) are managed via `.env` files and never exposed in the source code.
*   **Bot Protection:** Structured `robots.txt` and secure server actions ensure that only legitimate traffic interacts with the enrollment system.
